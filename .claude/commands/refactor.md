Проведи автоматический техрефакторинг всей базы знаний. Работай в четыре фазы.

Перед началом прочитай `structure.md` (правила документации), `templates/city.md` (шаблон статьи о городе) и `templates/walking-route.md` (шаблон пешеходного маршрута).

## Фаза 1: Подготовка

1. Создай новую ветку: `refactor/cleanup-ГГГГ-ММ-ДД` (текущая дата).
2. Собери список всех статей (`travel-knowledge-base/**/*.md`).

## Фаза 2: Анализ и исправление

Пройди по каждой статье и исправь проблемы в четырёх категориях:

### Стандартизация
- Единообразные уровни заголовков (H1 — название, H2 — разделы, H3 — подразделы)
- Единый формат таблиц: выравнивание разделителей, наличие заголовков
- Единообразное форматирование списков, ссылок, выделения текста

### Упрощение
- Удаление избыточного текста и повторов
- Сокращение многословных описаний без потери смысла
- Удаление пустых или бессодержательных секций

### Обобщение
- Приведение всех статей к правилам `structure.md`
- Статьи о городах: соответствие шаблону `templates/city.md` — наличие обязательных разделов, формат таблиц
- Пешеходные маршруты (папка `london-walking-routes/`): соответствие шаблону `templates/walking-route.md` — наличие обязательных разделов, формат таблиц
- Русский язык везде, кроме устоявшихся английских названий

### Мелкая реструктуризация
- Порядок секций строго по соответствующему шаблону: `templates/city.md` для городов, `templates/walking-route.md` для пешеходных маршрутов
- Добавление отсутствующих обязательных разделов (с пометкой «нет данных» если нужно)
- Удаление разделителей и элементов, не предусмотренных шаблоном

Веди лог изменений: для каждого файла фиксируй, что именно исправлено и в какой категории.

## Фаза 3: Коммит

1. Добавь все изменённые файлы в индекс.
2. Создай коммит с сообщением, описывающим суть изменений.

## Фаза 4: Pull Request

1. Запуш ветку: `git push -u origin HEAD`.
2. Создай PR через `gh pr create`:
   - **Заголовок**: описательный, например «Refactor: стандартизация форматирования и порядка секций»
   - **Тело PR**: перечисли все изменения, сгруппированные по четырём категориям (стандартизация, упрощение, обобщение, реструктуризация). Для каждого изменения укажи файл и суть правки.
